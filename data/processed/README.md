# Processed datasets

This directory contains modeling-ready datasets derived from the curated
antioxidant activity datasets located in `data/curated`.

These datasets represent branch-specific subsets and stratified datasets
used directly for QSAR model training.

# Dataset descriptions

## `antioxidant13.csv`

### How it is created

This dataset is generated from `antioxidant12.csv` by filtering the potency
category corresponding to **high antioxidant activity**:

Category = Bajo

Creation script: `ensayo58.py`

### What it represents

High-activity subset extracted from the IC50 processed dataset.

Lower IC50 values correspond to **higher antioxidant potency**, therefore
this dataset contains the most potent compounds within the dataset.

This dataset serves as the base dataset for **high-potency QSAR modeling branches**.

### Typical columns

- `Smiles`
- `Log10 Value (nM)`
- `Category`
- additional subcategory labels generated by downstream scripts

---

## `antioxidant14.csv`

### How it is created

This dataset is derived from `antioxidant13.csv`.

The high-activity dataset is further stratified into three potency bins using
quantile-based splitting.

Method used:

pandas.qcut(q=3)

Creation script: `ensayo75.py`

### What it represents

The high-activity dataset split into terciles to support
subcategory-specific QSAR modeling workflows.

---

## `antioxidant15.csv`

### How it is created

This dataset is generated from `antioxidant12.csv` by selecting the
**medium-activity subset**:

Category = Medio

The dataset is then stratified into four potency bins using
quantile-based splitting.

Method used:

pandas.qcut(q=4)

Creation script: `ensayo80.py`

### What it represents

Medium-activity IC50 dataset split into **four potency subcategories**
used for targeted QSAR model training.

### Key branching column

Subcategory

---

## `antioxidant18.csv`

### What it represents

This dataset corresponds to a later-stage processed dataset used by the
modeling workflow implemented in `ensayo171.py`.

Unlike datasets `13–15`, this dataset belongs to a **low-activity modeling branch**.

These compounds generally exhibit **higher IC50 values**, indicating
lower antioxidant potency.

### Confirmed usage

Used directly in the script:

`ensayo171.py`

# Role within the project pipeline

The datasets follow the pipeline:

data_acquisition → data/raw → data/curated → data/processed → models

Where:

- `data/raw` contains heterogeneous activity datasets obtained from external databases.
- `data/curated` contains progressively cleaned and normalized datasets.
- `data/processed` contains modeling-ready datasets used directly by QSAR models.
- `models` contains trained predictive models and evaluation results.
