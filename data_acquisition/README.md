# Data acquisition

This directory contains scripts for the acquisition and initial curation of molecular and
bioactivity data from public chemical databases, primarily ChEMBL.

The workflow is divided into two main stages: data retrieval and data filtering.

---

# Files

## Input Files

**`molecules.csv`**

Contains ChEMBL assay IDs in a single column (`aidextid`).

These identifiers are used by `parte2.py` to retrieve molecular and activity data from the ChEMBL API.

---

## Output Files

**`molecule_definitive.csv`**

Generated by `parte2.py`.  
Contains full activity-level and molecule-level information retrieved from the ChEMBL database.

**`molecule_filter.csv`**

Generated by `parte3.py`.  
Contains only rows where the `assay_description` field matches antioxidant-related keywords.

Filtered keywords:

- Antioxidant
- antioxidant
- lipid peroxidation

This dataset represents the initial antioxidant-focused subset used in the downstream
data processing pipeline.

---

# Source of assay identifiers

The file `molecules.csv` contains ChEMBL assay identifiers used as input for the retrieval pipeline.

These identifiers originate from prior database searches within ChEMBL targeting
bioassays associated with antioxidant activity and related biochemical processes.

---

# Part 2 — ChEMBL Data Retrieval

## Data Retrieved

### Activity-Level Information

- Standard activity type, relation, value, and units
- pChEMBL value
- Data validity and activity comments
- Ligand efficiency metrics (when available)

Assay metadata:

- Assay description
- Assay type
- BAO format and label

Target information:

- Target identifiers
- Target organism

Publication metadata:

- Document identifiers
- Journal name
- Publication year

---

### Molecule-Level Information

- ChEMBL molecule ID
- Preferred molecule name
- Development phase (max phase)

Chemical structure:

- Canonical SMILES

Physicochemical properties:

- Molecular weight
- ALogP
- Rule-of-five violations

---

# Workflow Overview

1. Load assay IDs from the input CSV file  
2. Validate and clean assay identifiers  
3. Query the ChEMBL Activity API for each assay  
4. Detect and parse API responses in JSON or XML format  
5. Extract activity records  
6. Retrieve detailed molecule information via the ChEMBL Molecule API  
7. Cache molecule data to avoid repeated API requests  
8. Merge activity-level and molecule-level information  
9. Append results incrementally to the output CSV file  

---

# API Handling and Robustness

The retrieval pipeline implements several mechanisms to ensure reliable data collection:

- Support for both JSON and XML API responses
- Automatic retry mechanism for failed API requests
- Request throttling to avoid exceeding API rate limits
- Local molecule-level cache to reduce redundant API calls
- Graceful handling of missing or incomplete data fields

# Part 3 — ChEMBL Antioxidant Assay Data Filtering

## Script Purpose

This script filters the dataset to retain only antioxidant-related assays.

The filtering step supports downstream data curation and QSAR modeling workflows.

---

## Filtering Criteria

Rows are kept only if the `assay_description` column contains at least one
of the following keywords:

- Antioxidant
- antioxidant
- lipid peroxidation

---

## Input and Output Files

### Input file

`molecule_definitive.csv`  
Dataset generated during the data acquisition stage.

### Output file

`molecule_filter.csv`  
Filtered dataset containing only antioxidant-related assays.
